PM Code — Research Synthesis

Key findings from UX, Architecture, and Devil's Advocate perspectives

The Core Tension

All three perspectives converged on one insight: the extension's value isn't making VS Code comfortable — it's making the bridge from "VS Code is open" to "my AI agent is doing useful work" as short as possible.

Top Findings by Agent

UX

UX Designer

Hybrid sidebar — native TreeView for navigation + WebviewView for search/detail panels. Zero JSON in the default path — forms for everything. Walkthroughs API for FTUE (5 steps max). Role-based presets to collapse time-to-first-value. Mental models from Notion/Figma/Jira should guide the UI vocabulary.

ARCH

Technical Architect

PM Code as MCP server — register itself via registerMcpServerDefinitionProvider() so any AI provider can consume skills/context natively. Provider adapter pattern for writing to each provider's config format. Layered state: settings.json for prefs, globalStorage for configs, SecretStorage for keys, globalState for UI state. Activation under 100ms.

DA

Devil's Advocate

Scope is 3 products in a trench coat. Cut to one killer feature: zero-config agent readiness — detect project + provider, install sane defaults, green checkmark in under 2 minutes. Static guides may be obsolete — the AI agent itself should be the guide. Client-agnostic is risky; VS Code + providers are building native MCP management. Rollback/undo is critically missing.

Consensus: What's Missing from the Brief

🔄 Rollback / Undo

Non-technical users WILL break configs. Need version history + "reset to working state"

👥 Team Sharing

Export/import config bundles, workspace-committed team profiles

🟢 Health Dashboard

Green/yellow/red status for all connectors. "Agent readiness" indicator.

💰 Cost Awareness

AI APIs cost money. PMs need usage tracking and spending alerts.

🔐 Credential Management

SecretStorage for API keys with guided setup. No raw tokens in JSON.

📁 Project Awareness

Different projects need different configs. Auto-detect project type + adapt.
