PM Code — Design Approaches

Three approaches to the C→B→A progression model. Roo Code first, zero VS Code familiarity assumed.

Each approach addresses: How does a PM who has never used VS Code go from install → productive AI agent use?

A

The Guided Shell

PM Code owns the entire first-run experience. On first launch, it takes over the editor area with a full-screen welcome panel that hides the file tree, terminal, and other VS Code chrome. A step-by-step wizard handles Roo Code setup, first connector, and first guided interaction. The sidebar is a "learning track" with progressive chapters. As users gain confidence, PM Code gradually reveals VS Code features (file explorer, terminal) with contextual explanations. Think: a "training wheels" layer over VS Code.

Pros

  • Lowest barrier to entry — shields users from ALL VS Code complexity
  • Strong C→B→A progression — literally reveals features over time
  • Cohesive, branded experience

Cons

  • Highest engineering effort — fighting VS Code's UX paradigm
  • Risk of "VS Code within VS Code" — users learn PM Code, not VS Code
  • Fragile — VS Code updates could break the shell
  • Users may feel patronized as they level up
B

The Companion Panel

PM Code is an always-present guide in the editor area. On first launch, it opens as a pinned editor panel (like a persistent README) with a welcome flow. It coexists with VS Code's native UI rather than hiding it. The sidebar handles navigation (skills, connectors, guides), while the main panel is a contextual companion — showing the right guide for what you're doing, surfacing connector status, and providing "what's this?" explanations for VS Code concepts as you encounter them. Roo Code chat lives in its own sidebar; PM Code's panel provides the scaffolding around it.

Pros

  • Works WITH VS Code, not against it — users learn real VS Code as they go
  • Lower engineering risk — uses standard webview panels + tree views
  • Natural C→B→A: panel shrinks from guide → dashboard → hidden as users grow
  • Doesn't fight Roo Code for sidebar space

Cons

  • More overwhelming on first launch — VS Code UI is still visible
  • Panel competes for editor real estate with actual files
  • Less "magical" first impression than the Guided Shell
C

The Smart Sidebar

Everything lives in the sidebar — no editor panels needed for the core flow. PM Code claims a sidebar position (like Roo Code does) with a rich webview sidebar. The FTUE is an inline walkthrough within the sidebar itself. Skills, connectors, guides, and status are all in a scrollable, search-filtered sidebar view. Clicking items opens lightweight editor panels only when needed (e.g., for multi-step guides with visuals). The sidebar is context-aware: it shows relevant guides based on what file or project you're in.

Pros

  • Smallest footprint — doesn't take over editor area
  • Familiar pattern — Roo Code already taught users that sidebars are where AI tools live
  • Context-aware sidebar can proactively surface help
  • Easiest to ignore for power users (A mode = just collapse the sidebar)

Cons

  • Sidebar space is narrow (300-400px) — hard to show rich guides, visuals, forms
  • FTUE in a narrow sidebar may feel cramped for users who need hand-holding
  • Two sidebars (PM Code + Roo Code) competing for attention
  • Complex forms (connector config) are painful in sidebar width

My Recommendation: B (The Companion Panel)

It strikes the best balance: enough presence to guide a total beginner without fighting VS Code or creating a parallel universe. The panel can be as prominent or subtle as the user needs — full welcome wizard on day 1, contextual dashboard by week 2, closed entirely by month 2. And it works with Roo Code's sidebar rather than competing with it.
