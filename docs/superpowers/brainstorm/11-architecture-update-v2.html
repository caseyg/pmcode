agentskills.io format, ~/.agents cross-compat, Roo MCP paths, revised commands

1. Skills Use the agentskills.io SKILL.md Format

Not JSON. Each skill is a directory with a SKILL.md (YAML frontmatter + markdown instructions).

Skill directory structure (agentskills.io spec)
idea-triage/
├── SKILL.md # YAML frontmatter + instructions (required)
├── scripts/ # Optional executable code
├── references/ # Optional supplementary docs
└── assets/ # Optional templates, data files
Example: idea-triage/SKILL.md
---
name: idea-triage
description: Evaluate, prioritize, and organize feature ideas from Jira, Aha!, or Monday. Use when sorting a backlog, preparing for planning, or reviewing incoming requests.
license: MIT
metadata:
author: pmcode
version: "1.0"
category: planning
connectors: jira aha monday
allowed-tools: Bash(git:*) Read
---
## Instructions
When the user asks to triage ideas, follow these steps...
Compatible with skills.sh

npx skills add pmcode/idea-triage installs to the right place. PM Code can also install/manage skills via its own UI.

Compatible with Claude Code plugins

Plugin manifest format aligns with Anthropic's plugin.json + skills/ directory convention. Can be published to Anthropic's plugin marketplace.

2. Where Skills Are Loaded From

PM Code reads skills from multiple locations, in priority order.

Skill loading order (highest priority first)
Priority Path Source 1 ./.pmcode/skills/ Project-local skills (workspace) 2 ./.agents/skills/ Cross-agent project skills (skills.sh convention) 3 ~/.pmcode/skills/ PM Code global skills 4 ~/.agents/skills/ Cross-agent global skills (skills.sh -g) 5 (bundled) Default skills shipped with the extension
If the same skill name appears in multiple locations, highest priority wins. Bundled defaults are overridden by any user-installed version.

3. Roo Code MCP Config — Actual Paths

PM Code writes MCP server definitions directly to these files.

Roo Code MCP configuration files (macOS)
Global (user-level)
~/Library/Application Support/Code/User/globalStorage/rooveterinaryinc.roo-cline/settings/cline_mcp_settings.json
Applies across all workspaces. PM Code writes here for global connectors.
Project-level (overrides global)
<project-root>/.roo/mcp.json
Committed to version control. Project-level overrides global for same server name.
Config format (both files)
{
  "mcpServers": {
    "jira": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-server-jira"],
      "env": {
        "JIRA_URL": "https://myteam.atlassian.net",
        "JIRA_API_TOKEN": "xxx"
      }
    }
  }
}
PM Code writes tokens inline to Roo's env

Tokens are written to the env block in Roo's MCP config. Also stored in ~/.pmcode/.env as the source of truth for PM Code's own reference.

PM Code reads before writing

Always reads existing config, merges PM Code's servers, preserves user-added servers. Never overwrites the whole file.
