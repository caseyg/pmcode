Sidebar does three things only

1. Search (takes over with results view)
2. Navigate (buttons open center panes)
3. Status (Roo connection, attention badges)

Clicking a button opens a center pane

Each item gets its own webview

Click "Sprint Retro" in the skills list → opens a dedicated Sprint Retro webview panel. Multiple skill panels can be open simultaneously as tabs.

2. ~/.pmcode — The Config Home

All PM Code state lives in one directory. Portable, inspectable, git-friendly.

~/.pmcode/ directory structure
~/.pmcode/
├── .env # credentials: API tokens, keys
├── config.json # user preferences, UI state, FTUE progress
├── memory/ # user memory (like CLAUDE.md learnings)
│ ├── preferences.md # how user likes to work
│ └── context.md # project-specific notes
├── skills/ # skill definitions
│ ├── idea-triage.json # bundled skill
│ ├── sprint-retro.json # bundled skill
│ ├── prd-writer.json # bundled skill
│ └── custom/ # user-created skills
├── connectors/ # connector definitions & state
│ ├── jira.json # config (non-secret settings)
│ ├── github.json
│ ├── monday.json
│ ├── aha.json
│ └── tavily.json
├── guides/ # guide progress & bookmarks
│ └── progress.json
└── history/ # config snapshots for rollback
├── 2026-03-30T10-00.json
└── 2026-03-30T11-30.json
Credentials in ~/.pmcode/.env

JIRA_API_TOKEN=xxx
MONDAY_API_TOKEN=xxx
TAVILY_API_KEY=xxx
Standard .env format. Skills and connectors read from here.

MCP config written directly to Roo

Connectors write their MCP server definitions directly to Roo's config files. Tokens referenced from .env or inlined per Roo's format.

User memory

~/.pmcode/memory/ stores learned preferences, project context, and behavioral notes — similar to how Claude Code uses CLAUDE.md. Persists across sessions.

Portable & inspectable

Everything is plain files. Users can back up, sync, share, or hand-edit if needed. No opaque VS Code globalState.

3. VS Code Command Architecture

Every screen and action is a VS Code command. Enables agent automation, keyboard shortcuts, and the command palette.

Command Registry
Navigation
pmcode.openSkills Open Skills list panel
pmcode.openConnectors Open Connectors list panel
pmcode.openGuides Open Guides list panel
pmcode.openSkill {id} Open specific skill detail panel
pmcode.openConnector {id} Open specific connector config panel
pmcode.openGuide {id} Open specific guide panel
pmcode.openDashboard Open companion dashboard
pmcode.openSettings Open PM Code settings
Connector Actions
pmcode.connector.install {id} Install a connector
pmcode.connector.configure {id} Open connector config form
pmcode.connector.test {id} Test connector health
pmcode.connector.enable {id} Enable a connector
pmcode.connector.disable {id} Disable a connector
pmcode.connector.remove {id} Remove a connector
Skill Actions
pmcode.skill.install {id} Install a skill
pmcode.skill.remove {id} Remove a skill
pmcode.skill.run {id} Run skill (inject prompt into Roo)
pmcode.skill.sendPrompt {text} Send arbitrary prompt to Roo
Guide Actions
pmcode.guide.start {id} Start or resume a guide
pmcode.guide.completeStep {id} {step} Mark step complete
pmcode.guide.reset {id} Reset guide progress
System
pmcode.checkDependencies Run system dependency check
pmcode.healthCheck Check all connector health
pmcode.rollback Rollback to previous config state
pmcode.resetFTUE Reset first-time experience
pmcode.focusSidebar Focus the PM Code sidebar
pmcode.search {query} Open sidebar with search pre-filled
Agent-friendly

Roo can call pmcode.connector.test jira to verify a connector works, or pmcode.openGuide sharing-context to help a user. Full programmatic control.

Command palette works

All commands appear in Cmd+Shift+P with human-readable titles. Power users (mode A) can access everything without the sidebar.
